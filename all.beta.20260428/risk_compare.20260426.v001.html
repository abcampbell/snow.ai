How risk is calculated (read once, internalize)
  1. Per-position greeks — options_risk.PortfolioRisk._compute_greeks. STK/ETF: δ=1, δ$=qty×price. FUT: δ=1, δ$=qty×price×mult (cents→$ for grains/cattle/sugar via PRICE_MAGNIFIER). OPT: solve IV from observed premium via Black-Scholes, then δ$=qty·N(d1)·S·mult. FOP: same with Black-76 on F.
  2. Underlying spot — first STK/FUT in book → prices[sym]. For FOPs whose future isn't in the book (SB, LE, ZW), _fetch_yf_spot() uses LE=F, SB=F, ZW=F, CL=F, GBPUSD=X (and divides yfinance =F prices by 100 since they come back in cents).
  3. IV source — Black-Scholes/Black-76 solver against observed market premium. Fallback 30% flat when premium is zero AND F is known. NOT realized-vol blend. IB's modelGreeks.impliedVol would be available with live data + genericTickList="106"; right now TWS is in delayed-frozen mode so greeks are stripped.
  4. Portfolio variance — portfolio_risk_decomp.py: yfinance 1y daily returns → annualized cov Σ. Weights w_i = (Σ δ$_j) / NLV per underlying. σ²_p = w'Σw. Daily VaR = σ_p·NLV / √252. Marginal risk per asset = w_i·(Σw)_i / σ_p.
  5. Vol-aware scenarios — risk_convexity.py: full reprice with delta + gamma + vega + theta under named thesis scenarios. Asymmetry = max(±30% reprice) / min(±30% reprice). Convex long when up>down, convex short when down>up.
Known accuracy gaps (in plain text)
  • 9984.T currency residue — Softbank position still shows ~$3k of phantom delta (JPY treated as USD). Tiny in absolute, but it's there.
  • Option premium = 0 fallback — when JSON has marketPrice=0 for an option (illiquid Friday close), we use 30% IV. Overstates ITM, understates deep OTM. Most affected: 4 sugar strikes (SB qty=2/-6/6/2) where IV=0.3 fallback fired.
  • Delta-normal VaR — gamma/vega contribution to daily VaR is not in the $1,719 number. A vol shock in ags or credit could move +/-10% of NLV that won't show up in the linear VaR — only in the named scenarios.
  • 1-year covariance lookback — captures recent regime, misses 2022-style shifts. Conservative for current vol, optimistic for tail.
  • portfolio_risk_decomp.py uses ETF proxies (CORN, CANE, WEAT) for futures — these track imperfectly vs the actual front-month. SONIA → TLT proxy is the biggest stretch and probably overstates correlation between SONIA and IEI.
